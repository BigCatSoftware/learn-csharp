# 1D Dynamic Programming

*Single-dimensional state problems and classic patterns*

In 1D DP, the state is defined by a single parameter — typically an index into an array or a remaining amount. The DP table is a one-dimensional array where `dp[i]` encodes the answer to a subproblem of size or position `i`.

## Common 1D Patterns

| Pattern | State Definition | Example |
|---------|-----------------|---------|
| Linear scan | `dp[i]` = answer considering first `i` elements | House Robber |
| Subsequence | `dp[i]` = answer ending at index `i` | Longest Increasing Subsequence |
| Target building | `dp[i]` = ways/min to reach value `i` | Coin Change, Word Break |
| Decision at each step | `dp[i]` = best result with choice at step `i` | Decode Ways |

## State Definition Tips

- **"ending at i"** vs **"considering first i elements"** — these lead to different recurrences. Choose based on the problem.
- If the recurrence only looks back a fixed number of steps, space can be optimized to O(1).
- If `dp[i]` depends on all previous states, the inner loop makes it O(n^2).

## Practice Problems

### Problem 1: House Robber (LeetCode 198)

**Problem:** Given an array of non-negative integers representing money at each house, find the maximum amount you can rob without robbing two adjacent houses.

**State:** `dp[i]` = maximum money robbing from houses `0..i`

**Recurrence:** `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
- - Skip house `i` — take `dp[i-1]`
- - Rob house `i` — take `dp[i-2] + nums[i]`


**Base cases:** `dp[0] = nums[0]`, `dp[1] = max(nums[0], nums[1])`

```csharp
public class Solution
{
    public int Rob(int[] nums)
    {
        int n = nums.Length;
        if (n == 1) return nums[0];

        // Space-optimized: only need two previous values
        int prev2 = nums[0];
        int prev1 = Math.Max(nums[0], nums[1]);

        for (int i = 2; i < n; i++)
        {
            int curr = Math.Max(prev1, prev2 + nums[i]);
            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;
    }
}
```

**Complexity:** O(n) time, O(1) space.

### Problem 2: House Robber II (LeetCode 213)

**Problem:** Same as House Robber, but the houses are arranged in a circle (the first and last houses are adjacent).

**Approach:** If we rob house 0, we cannot rob house n-1, and vice versa. Run House Robber twice:
1. On `nums[0..n-2]` (include first, exclude last)
2. On `nums[1..n-1]` (exclude first, include last)

Take the maximum of both.

```csharp
public class Solution
{
    public int Rob(int[] nums)
    {
        int n = nums.Length;
        if (n == 1) return nums[0];
        if (n == 2) return Math.Max(nums[0], nums[1]);

        // Rob houses 0..n-2 or houses 1..n-1
        return Math.Max(
            RobLinear(nums, 0, n - 2),
            RobLinear(nums, 1, n - 1)
        );
    }

    private int RobLinear(int[] nums, int start, int end)
    {
        int prev2 = 0, prev1 = 0;

        for (int i = start; i <= end; i++)
        {
            int curr = Math.Max(prev1, prev2 + nums[i]);
            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;
    }
}
```

**Complexity:** O(n) time, O(1) space.

### Problem 3: Longest Increasing Subsequence (LeetCode 300)

**Problem:** Given an integer array `nums`, return the length of the longest strictly increasing subsequence.

#### Approach 1: O(n^2) DP

**State:** `dp[i]` = length of the LIS ending at index `i`

**Recurrence:** `dp[i] = max(dp[j] + 1)` for all `j < i` where `nums[j] < nums[i]`

**Base case:** `dp[i] = 1` (every element is a subsequence of length 1)

```csharp
public class Solution
{
    public int LengthOfLIS_N2(int[] nums)
    {
        int n = nums.Length;
        var dp = new int[n];
        Array.Fill(dp, 1);

        int maxLen = 1;

        for (int i = 1; i < n; i++)
        {
            for (int j = 0; j < i; j++)
            {
                if (nums[j] < nums[i])
                    dp[i] = Math.Max(dp[i], dp[j] + 1);
            }
            maxLen = Math.Max(maxLen, dp[i]);
        }

        return maxLen;
    }
}
```

**Complexity:** O(n^2) time, O(n) space.

#### Approach 2: O(n log n) with Patience Sorting

Maintain a list `tails` where `tails[i]` is the smallest tail element for an increasing subsequence of length `i+1`. Use binary search to find the insertion position.

```csharp
public class Solution
{
    public int LengthOfLIS(int[] nums)
    {
        var tails = new List<int>();

        foreach (int num in nums)
        {
            // Binary search for the first element >= num
            int lo = 0, hi = tails.Count;
            while (lo < hi)
            {
                int mid = lo + (hi - lo) / 2;
                if (tails[mid] < num) lo = mid + 1;
                else hi = mid;
            }

            if (lo == tails.Count)
                tails.Add(num);     // extend the longest subsequence
            else
                tails[lo] = num;    // replace to keep the smallest tail
        }

        return tails.Count;
    }
}
```

**Complexity:** O(n log n) time, O(n) space.

> **Note:** The `tails` list is NOT the actual LIS — it tracks the smallest possible tail for each length. To reconstruct the LIS, you need additional bookkeeping.

### Problem 4: Word Break (LeetCode 139)

**Problem:** Given a string `s` and a dictionary of words, return `true` if `s` can be segmented into a space-separated sequence of dictionary words.

**State:** `dp[i]` = true if `s[0..i-1]` can be segmented

**Recurrence:** `dp[i] = true` if there exists some `j < i` such that `dp[j] == true` and `s[j..i-1]` is in the dictionary

**Base case:** `dp[0] = true` (empty string can be segmented)

```csharp
public class Solution
{
    public bool WordBreak(string s, IList<string> wordDict)
    {
        var wordSet = new HashSet<string>(wordDict);
        int n = s.Length;
        var dp = new bool[n + 1];
        dp[0] = true; // empty string

        for (int i = 1; i <= n; i++)
        {
            for (int j = 0; j < i; j++)
            {
                // If s[0..j-1] is breakable and s[j..i-1] is a word
                if (dp[j] && wordSet.Contains(s.Substring(j, i - j)))
                {
                    dp[i] = true;
                    break; // no need to check further
                }
            }
        }

        return dp[n];
    }
}
```

**Complexity:** O(n^2 · k) time where k is average word length (for substring hashing), O(n) space.

### Problem 5: Decode Ways (LeetCode 91)

**Problem:** A message of digits can be decoded into letters (A=1, B=2, ..., Z=26). Given a string of digits, return the number of ways to decode it.

**State:** `dp[i]` = number of ways to decode `s[0..i-1]`

**Recurrence:**
- If `s[i-1] != '0'`: `dp[i] += dp[i-1]` (single digit decode)
- If `s[i-2..i-1]` forms 10–26: `dp[i] += dp[i-2]` (two digit decode)

**Base cases:** `dp[0] = 1`, `dp[1] = (s[0] != '0') ? 1 : 0`

```csharp
public class Solution
{
    public int NumDecodings(string s)
    {
        int n = s.Length;
        if (n == 0 || s[0] == '0') return 0;

        // Space-optimized
        int prev2 = 1; // dp[0] — empty prefix
        int prev1 = 1; // dp[1] — first char (we checked it's not '0')

        for (int i = 2; i <= n; i++)
        {
            int curr = 0;

            // Single digit: s[i-1]
            if (s[i - 1] != '0')
                curr += prev1;

            // Two digits: s[i-2..i-1]
            int twoDigit = (s[i - 2] - '0') * 10 + (s[i - 1] - '0');
            if (twoDigit >= 10 && twoDigit <= 26)
                curr += prev2;

            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;
    }
}
```

**Complexity:** O(n) time, O(1) space.

### Problem 6: Coin Change (LeetCode 322)

**Problem:** Given coin denominations and a target amount, find the minimum number of coins needed. Return -1 if impossible.

**State:** `dp[i]` = minimum coins to make amount `i`

**Recurrence:** `dp[i] = min(dp[i - coin] + 1)` for each coin where `coin <= i`

**Base case:** `dp[0] = 0` (zero coins for amount 0)

```csharp
public class Solution
{
    public int CoinChange(int[] coins, int amount)
    {
        var dp = new int[amount + 1];
        Array.Fill(dp, amount + 1); // use amount+1 as "infinity"
        dp[0] = 0;

        for (int i = 1; i <= amount; i++)
        {
            foreach (int coin in coins)
            {
                if (coin <= i)
                    dp[i] = Math.Min(dp[i], dp[i - coin] + 1);
            }
        }

        return dp[amount] > amount ? -1 : dp[amount];
    }
}
```

**Complexity:** O(amount * coins) time, O(amount) space.

## Summary of Recurrences

| Problem | Recurrence | Time | Space |
|---------|-----------|------|-------|
| House Robber | `dp[i] = max(dp[i-1], dp[i-2]+nums[i])` | O(n) | O(1) |
| House Robber II | Two passes of House Robber | O(n) | O(1) |
| LIS (DP) | `dp[i] = max(dp[j]+1)` for `j<i`, `nums[j]<nums[i]` | O(n^2) | O(n) |
| LIS (Binary Search) | Patience sorting with binary search | O(n log n) | O(n) |
| Word Break | `dp[i] = any dp[j] && s[j..i] in dict` | O(n^2) | O(n) |
| Decode Ways | `dp[i] = dp[i-1] (if valid) + dp[i-2] (if valid)` | O(n) | O(1) |
| Coin Change | `dp[i] = min(dp[i-coin]+1)` | O(n*c) | O(n) |

## Common Mistakes

1. **Forgetting the '0' edge cases in Decode Ways** — `"0"`, `"30"`, and `"100"` need special handling.
2. **Using `int.MaxValue` as infinity then adding 1** — This overflows. Use `amount + 1` or check before adding.
3. **LIS: confusing subsequence with subarray** — A subsequence does not need to be contiguous.
4. **Word Break: not using a HashSet** — Linear search through the word list for each substring is too slow.

## Interview Tips

- For House Robber variants, always think: "What is the last decision?" — rob or skip.
- For LIS, mention both O(n^2) and O(n log n) approaches. Implement O(n^2) first if time is short.
- For Coin Change, clarify: is it minimum coins (this problem) or number of ways (Coin Change II — that is a knapsack)?
- Always start by writing the recurrence on paper before coding.

## Quiz

<details>
<summary>1. Why can House Robber be solved in O(1) space?</summary>

Because `dp[i]` only depends on `dp[i-1]` and `dp[i-2]`. We only need two variables to track the previous two states, not the entire array.
</details>

<details>
<summary>2. What makes the O(n log n) LIS solution faster than the O(n^2) one?</summary>

Instead of scanning all previous indices for each element, it maintains a sorted `tails` array and uses binary search (O(log n)) to find the correct insertion position. This replaces the inner O(n) loop with O(log n).
</details>

<details>
<summary>3. In Word Break, why do we use dp[0] = true?</summary>

`dp[0]` represents the empty string, which is trivially segmentable. It serves as the anchor: when we find that `s[0..i-1]` is a dictionary word, we need `dp[0] = true` to mark it as valid.
</details>

<details>
<summary>4. In Coin Change, why do we initialize with amount+1 instead of int.MaxValue?</summary>

Because we add 1 in the recurrence (`dp[i - coin] + 1`). If we used `int.MaxValue`, adding 1 would cause integer overflow. Using `amount + 1` is safe and is always larger than any valid answer (you can never need more than `amount` coins if the smallest coin is 1).
</details>

<details>
<summary>5. How does House Robber II reduce to House Robber?</summary>

In a circle, house 0 and house n-1 are adjacent, so we cannot rob both. We solve two linear subproblems: rob houses 0 to n-2 (exclude last) and rob houses 1 to n-1 (exclude first). The answer is the maximum of both.
</details>
