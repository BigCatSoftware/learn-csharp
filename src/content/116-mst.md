# Minimum Spanning Trees

## What Is a Minimum Spanning Tree?

A **Minimum Spanning Tree (MST)** of a weighted, connected, undirected graph is a subset of edges that connects all vertices with the minimum total edge weight and no cycles. An MST has exactly `V - 1` edges for `V` vertices.

- Example graph: — MST (total weight = 10):
- A ---4--- B — A ---4--- B
- | — \ — | — | — |
- 2 — 3 — 5 — 2 — 5
- | — \ — | — | — |
- C ---1--- D — C ---1--- D
- All edges: A-B(4), A-C(2), A-D(3), B-D(5), C-D(1)
- MST edges: C-D(1), A-C(2), A-B(4) — wait, that's weight 7
- Actually let's recalculate:
- Sorted: C-D(1), A-C(2), A-D(3), A-B(4), B-D(5)
- Pick C-D(1): connects C,D — total=1
- Pick A-C(2): connects A to {C,D} — total=3
- Pick A-D(3): skip, A and D already connected
- Pick A-B(4): connects B — total=7
- MST weight = 7, edges = {C-D, A-C, A-B}
- A ---4--- B
- C ---1--- D


## Kruskal's Algorithm (Sort Edges + Union-Find)

**Strategy:** Sort all edges by weight. Greedily add the smallest edge that doesn't create a cycle. Use **Union-Find** to efficiently detect cycles.

- Kruskal's step-by-step on the example above:
- Sorted edges: C-D(1), A-C(2), A-D(3), A-B(4), B-D(5)
- Step 1: C-D(1) — add (no cycle) — Components: {A}, {B}, {C,D}
- Step 2: A-C(2) — add (no cycle) — Components: {A,C,D}, {B}
- Step 3: A-D(3) — skip (A,D same component — would create cycle)
- Step 4: A-B(4) — add (no cycle) — Components: {A,B,C,D}
- V-1 = 3 edges added — DONE
- MST weight = 1 + 2 + 4 = 7


### Union-Find Implementation

```csharp
public class UnionFind
{
    private int[] parent;
    private int[] rank;

    public UnionFind(int n)
    {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++)
            parent[i] = i; // each node is its own parent
    }

    // Find with path compression
    public int Find(int x)
    {
        if (parent[x] != x)
            parent[x] = Find(parent[x]); // path compression
        return parent[x];
    }

    // Union by rank — returns false if already connected
    public bool Union(int x, int y)
    {
        int rootX = Find(x), rootY = Find(y);
        if (rootX == rootY) return false; // already connected (cycle!)

        if (rank[rootX] < rank[rootY])
            parent[rootX] = rootY;
        else if (rank[rootX] > rank[rootY])
            parent[rootY] = rootX;
        else
        {
            parent[rootY] = rootX;
            rank[rootX]++;
        }
        return true;
    }
}
```

### Kruskal's Implementation

```csharp
public class KruskalMST
{
    public int FindMST(int n, int[][] edges)
    {
        // edges[i] = [u, v, weight]
        // Sort edges by weight
        Array.Sort(edges, (a, b) => a[2].CompareTo(b[2]));

        var uf = new UnionFind(n);
        int mstWeight = 0;
        int edgesUsed = 0;

        foreach (var edge in edges)
        {
            int u = edge[0], v = edge[1], w = edge[2];

            if (uf.Union(u, v)) // no cycle — add to MST
            {
                mstWeight += w;
                edgesUsed++;

                if (edgesUsed == n - 1) break; // MST complete
            }
        }

        // If edgesUsed < n-1, graph is disconnected (no MST exists)
        return edgesUsed == n - 1 ? mstWeight : -1;
    }
}
```

## Prim's Algorithm (Greedy with Min-Heap)

**Strategy:** Start from any vertex. Greedily add the cheapest edge connecting a visited vertex to an unvisited vertex. Use a **min-heap (priority queue)** for efficiency.

```
Prim's step-by-step starting from A:

Start: visited = {A}
  Edges from A: A-B(4), A-C(2), A-D(3) → heap: [(2,C), (3,D), (4,B)]

Step 1: Pop (2,C) → add C, weight=2, visited={A,C}
  New edges from C: C-D(1) → heap: [(1,D), (3,D), (4,B)]

Step 2: Pop (1,D) → add D, weight=1, visited={A,C,D}
  New edges from D: D-B(5) → heap: [(3,D), (4,B), (5,B)]

Step 3: Pop (3,D) → D already visited, skip
Step 4: Pop (4,B) → add B, weight=4, visited={A,B,C,D}
  All vertices visited → DONE

MST weight = 2 + 1 + 4 = 7
```

### Prim's Implementation

```csharp
public class PrimMST
{
    public int FindMST(int n, int[][] edges)
    {
        // Build adjacency list
        var adj = new List<(int node, int weight)>[n];
        for (int i = 0; i < n; i++)
            adj[i] = new List<(int, int)>();

        foreach (var edge in edges)
        {
            adj[edge[0]].Add((edge[1], edge[2]));
            adj[edge[1]].Add((edge[0], edge[2]));
        }

        // Min-heap: (weight, node)
        var heap = new PriorityQueue<int, int>(); // node, priority=weight
        bool[] visited = new bool[n];
        int mstWeight = 0;
        int edgesUsed = 0;

        // Start from node 0
        heap.Enqueue(0, 0); // (node=0, weight=0)

        while (heap.Count > 0 && edgesUsed < n)
        {
            heap.TryDequeue(out int node, out int weight);

            if (visited[node]) continue; // already in MST

            visited[node] = true;
            mstWeight += weight;
            edgesUsed++;

            // Add all edges from this node
            foreach (var (neighbor, edgeWeight) in adj[node])
            {
                if (!visited[neighbor])
                    heap.Enqueue(neighbor, edgeWeight);
            }
        }

        return edgesUsed == n ? mstWeight : -1;
    }
}
```

## Kruskal's vs Prim's

| Aspect | Kruskal's | Prim's |
|---|---|---|
| Approach | Sort edges, add smallest safe edge | Grow tree from a vertex, add cheapest connection |
| Data structure | Union-Find | Min-Heap + visited array |
| Time complexity | O(E log E) | O(E log V) with binary heap |
| Best for | Sparse graphs (few edges) | Dense graphs (many edges) |
| Edge list format | Natural (works on edge list) | Needs adjacency list |
| Can detect disconnected graph | Yes (edgesUsed < V-1) | Yes (edgesUsed < V) |

## Practice Problems

### Problem 1: Min Cost to Connect All Points

**Statement:** Given an array of `points` where `points[i] = [xi, yi]`, return the minimum cost to connect all points. The cost to connect two points is the Manhattan distance: `|xi - xj| + |yi - yj|`. (This is essentially finding the MST of a complete graph.)

**Approach (Prim's):** For a complete graph with n points, there are O(n^2) edges. Prim's is preferred here because we can lazily generate edges.

```csharp
public class Solution
{
    public int MinCostConnectPoints(int[][] points)
    {
        int n = points.Length;
        bool[] visited = new bool[n];

        // Min-heap: (cost, pointIndex)
        var heap = new PriorityQueue<int, int>();
        heap.Enqueue(0, 0); // start from point 0 with cost 0

        int totalCost = 0;
        int connected = 0;

        while (connected < n)
        {
            heap.TryDequeue(out int node, out int cost);

            if (visited[node]) continue;

            visited[node] = true;
            totalCost += cost;
            connected++;

            // Add edges to all unvisited points
            for (int j = 0; j < n; j++)
            {
                if (!visited[j])
                {
                    int dist = Math.Abs(points[node][0] - points[j][0])
                             + Math.Abs(points[node][1] - points[j][1]);
                    heap.Enqueue(j, dist);
                }
            }
        }

        return totalCost;
    }
}
```

**Alternative (Kruskal's):**

```csharp
public class Solution
{
    public int MinCostConnectPointsKruskal(int[][] points)
    {
        int n = points.Length;

        // Generate all edges
        var edges = new List<int[]>(); // [cost, i, j]
        for (int i = 0; i < n; i++)
            for (int j = i + 1; j < n; j++)
            {
                int dist = Math.Abs(points[i][0] - points[j][0])
                         + Math.Abs(points[i][1] - points[j][1]);
                edges.Add(new[] { dist, i, j });
            }

        // Sort by cost
        edges.Sort((a, b) => a[0].CompareTo(b[0]));

        var uf = new UnionFind(n);
        int totalCost = 0;
        int edgesUsed = 0;

        foreach (var edge in edges)
        {
            if (uf.Union(edge[1], edge[2]))
            {
                totalCost += edge[0];
                edgesUsed++;
                if (edgesUsed == n - 1) break;
            }
        }

        return totalCost;
    }
}
```

**Complexity:** Prim's: O(n^2 log n) time, O(n^2) heap space. Kruskal's: O(n^2 log n) time, O(n^2) space for edge list.

### Problem 2: MST Weight Calculation (Generic)

**Statement:** Given `n` nodes and a list of weighted edges, compute the MST weight. If the graph is disconnected, return -1.

**Approach:** Use Kruskal's for simplicity. This is the generic MST template.

```csharp
public class Solution
{
    public int MinimumSpanningTreeWeight(int n, int[][] edges)
    {
        // Sort edges by weight
        Array.Sort(edges, (a, b) => a[2].CompareTo(b[2]));

        var uf = new UnionFind(n);
        int mstWeight = 0;
        int components = n; // start with n components

        foreach (var edge in edges)
        {
            int u = edge[0], v = edge[1], w = edge[2];

            if (uf.Union(u, v))
            {
                mstWeight += w;
                components--;

                if (components == 1) break; // fully connected
            }
        }

        return components == 1 ? mstWeight : -1; // -1 if disconnected
    }
}

// Reusable Union-Find
public class UnionFind
{
    private int[] parent, rank;

    public UnionFind(int n)
    {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    public int Find(int x)
    {
        if (parent[x] != x)
            parent[x] = Find(parent[x]);
        return parent[x];
    }

    public bool Union(int x, int y)
    {
        int rx = Find(x), ry = Find(y);
        if (rx == ry) return false;
        if (rank[rx] < rank[ry]) parent[rx] = ry;
        else if (rank[rx] > rank[ry]) parent[ry] = rx;
        else { parent[ry] = rx; rank[rx]++; }
        return true;
    }
}
```

**Complexity:** Time O(E log E + E * alpha(V)) where alpha is the inverse Ackermann function (effectively O(1)). Space O(V + E).

## Common Mistakes

1. **Forgetting path compression in Union-Find.** Without it, `Find` can degrade to O(n). Always include `parent[x] = Find(parent[x])`.
2. **Not handling disconnected graphs.** Always check that you added exactly `V-1` edges. If not, the graph has no spanning tree.
3. **Prim's: not checking visited before processing.** Stale entries in the heap can process already-visited nodes, leading to incorrect results.
4. **Edge direction in undirected graphs.** When building an adjacency list, add edges in both directions: `adj[u].Add(v)` AND `adj[v].Add(u)`.
5. **Using Kruskal's on dense graphs.** Generating all O(n^2) edges for complete graphs uses O(n^2) memory. Prim's with an adjacency approach may be more practical.

## Interview Tips

- **Know both algorithms** and when each is better: Kruskal's for sparse (edge list given), Prim's for dense (adjacency list/matrix).
- **Union-Find is reusable.** Practice it separately — it appears in MST, connected components, cycle detection, and many other problems.
- **Explain the greedy proof:** "Both algorithms are greedy and correct by the cut property: the lightest edge crossing any cut must be in some MST."
- **Time complexity matters:** State "O(E log E) for Kruskal's" or "O(E log V) for Prim's" confidently.
- **Edge case:** A graph with 1 node has MST weight 0 and 0 edges.

## Quiz

<details>
<summary>1. What is the "cut property" and why does it guarantee MST correctness?</summary>

The cut property states: for any cut (partition of vertices into two sets), the minimum weight edge crossing the cut must be part of some MST. Both Kruskal's and Prim's algorithms rely on this property. Kruskal's finds the globally cheapest safe edge; Prim's finds the cheapest edge crossing the cut between visited and unvisited vertices. In both cases, the cut property guarantees the chosen edge belongs to an MST.
</details>

<details>
<summary>2. What is path compression in Union-Find, and what is its amortized time complexity?</summary>

Path compression makes every node on the find path point directly to the root: `parent[x] = Find(parent[x])`. Combined with union by rank, this gives an amortized time complexity of O(alpha(n)) per operation, where alpha is the inverse Ackermann function — effectively O(1) for all practical input sizes (alpha(n) <= 4 for n up to 10^80).
</details>

<details>
<summary>3. Why is Kruskal's preferred for sparse graphs and Prim's for dense graphs?</summary>

Kruskal's sorts all edges: O(E log E). For sparse graphs (E close to V), this is O(V log V). For dense graphs (E close to V^2), sorting is O(V^2 log V). Prim's with a binary heap runs in O(E log V). For dense graphs, this is O(V^2 log V) — but Prim's with an adjacency matrix (no heap) runs in O(V^2), which beats Kruskal's O(V^2 log V) for dense graphs.
</details>

<details>
<summary>4. Can a graph have multiple valid MSTs? When does this happen?</summary>

Yes, a graph can have multiple MSTs if there are edges with equal weights. When two or more edges have the same weight and either could be chosen at a given step without creating a cycle, different choices lead to different spanning trees — but all have the same total weight. If all edge weights are distinct, the MST is unique.
</details>

<details>
<summary>5. In "Min Cost to Connect All Points", why might Prim's be preferred over Kruskal's?</summary>

With n points, the complete graph has n*(n-1)/2 edges. Kruskal's requires generating and sorting all O(n^2) edges upfront, using O(n^2) memory. Prim's can lazily generate edges by computing Manhattan distances on the fly as vertices are visited, and the heap typically holds far fewer than n^2 entries. For very large n, the memory savings of Prim's lazy approach can be significant. Additionally, an optimized Prim's without a heap (O(n^2) with an adjacency matrix scan) avoids the log factor entirely.
</details>
