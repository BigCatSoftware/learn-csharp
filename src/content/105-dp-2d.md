# 2D Dynamic Programming

*State tables with two dimensions for grid and sequence problems*

In 2D DP, the state depends on two parameters — often two indices, a position in a grid, or two sequence positions. The DP table is a 2D array `dp[i][j]` and the recurrence draws from neighboring cells.

## When to Use 2D DP

- Problems involving **two sequences** (LCS, Edit Distance, Interleaving String)
- **Grid traversal** problems (Unique Paths, minimum path sum)
- Problems where the state requires **two parameters** to fully describe a subproblem

## 2D State Table Visualization

For a grid problem like Unique Paths on a 3x4 grid:

- dp[i][j] = number of ways to reach cell (i,j)
- j=0 — j=1 — j=2 — j=3
- i=0 [ 1 — 1 — 1 — 1 — ] — only move right
- i=1 [ 1 — 2 — 3 — 4 — ]
- i=2 [ 1 — 3 — 6 — 10 — ] — dp[2][3] = 10 (answer)
- only move down
- dp[i][j] = dp[i-1][j] + dp[i][j-1]
- (from above) — (from left)


## Common 2D Patterns

| Pattern | Dimensions | Example |
|---------|-----------|---------|
| Grid paths | row × col | Unique Paths |
| Two sequences | len(s1) × len(s2) | LCS, Edit Distance |
| Subset + capacity | items × capacity | Knapsack (see lesson 106) |
| String + target | len(s) × target | Target Sum |

## Practice Problems

### Problem 1: Unique Paths (LeetCode 62)

**Problem:** A robot is at the top-left corner of an `m x n` grid. It can only move right or down. How many unique paths exist to the bottom-right corner?

**State:** `dp[i][j]` = number of unique paths to cell `(i, j)`

**Recurrence:** `dp[i][j] = dp[i-1][j] + dp[i][j-1]`

**Base cases:** `dp[0][j] = 1`, `dp[i][0] = 1` (only one way along edges)

```csharp
public class Solution
{
    public int UniquePaths(int m, int n)
    {
        var dp = new int[m, n];

        // Base cases: first row and first column are all 1
        for (int j = 0; j < n; j++) dp[0, j] = 1;
        for (int i = 0; i < m; i++) dp[i, 0] = 1;

        // Fill the table
        for (int i = 1; i < m; i++)
        {
            for (int j = 1; j < n; j++)
            {
                dp[i, j] = dp[i - 1, j] + dp[i, j - 1];
            }
        }

        return dp[m - 1, n - 1];
    }

    // Space-optimized: O(n) using a single row
    public int UniquePathsOptimized(int m, int n)
    {
        var dp = new int[n];
        Array.Fill(dp, 1); // first row is all 1s

        for (int i = 1; i < m; i++)
        {
            for (int j = 1; j < n; j++)
            {
                dp[j] += dp[j - 1]; // dp[j] (from above) + dp[j-1] (from left)
            }
        }

        return dp[n - 1];
    }
}
```

**Complexity:** O(m * n) time, O(n) space (optimized).

### Problem 2: Longest Common Subsequence (LeetCode 1143)

**Problem:** Given two strings `text1` and `text2`, return the length of their longest common subsequence.

**State:** `dp[i][j]` = LCS length of `text1[0..i-1]` and `text2[0..j-1]`

**Recurrence:**
- If `text1[i-1] == text2[j-1]`: `dp[i][j] = dp[i-1][j-1] + 1`
- Else: `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`

**Base cases:** `dp[0][j] = 0`, `dp[i][0] = 0` (empty string has LCS 0)

> Example: text1 = "abcde", text2 = "ace"
> "" | a | c | e
> "" | [ 0 | 0 | 0 | 0 ]
> a | [ 0 | 1 | 1 | 1 ]
> b | [ 0 | 1 | 1 | 1 ]
> c | [ 0 | 1 | 2 | 2 ]
> d | [ 0 | 1 | 2 | 2 ]
> e | [ 0 | 1 | 2 | 3 ] | answer = 3 ("ace")


```csharp
public class Solution
{
    public int LongestCommonSubsequence(string text1, string text2)
    {
        int m = text1.Length, n = text2.Length;
        var dp = new int[m + 1, n + 1];

        for (int i = 1; i <= m; i++)
        {
            for (int j = 1; j <= n; j++)
            {
                if (text1[i - 1] == text2[j - 1])
                    dp[i, j] = dp[i - 1, j - 1] + 1;
                else
                    dp[i, j] = Math.Max(dp[i - 1, j], dp[i, j - 1]);
            }
        }

        return dp[m, n];
    }
}
```

**Complexity:** O(m * n) time, O(m * n) space. Can be optimized to O(min(m, n)) space with rolling array.

### Problem 3: Edit Distance / Levenshtein Distance (LeetCode 72)

**Problem:** Given two strings `word1` and `word2`, return the minimum number of operations (insert, delete, replace) to convert `word1` into `word2`.

**State:** `dp[i][j]` = minimum operations to convert `word1[0..i-1]` to `word2[0..j-1]`

**Recurrence:**
- If `word1[i-1] == word2[j-1]`: `dp[i][j] = dp[i-1][j-1]` (no operation)
- Else: `dp[i][j] = 1 + min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1])`
```text
  - `dp[i-1][j-1] + 1` → replace
  - `dp[i-1][j] + 1` → delete from word1
  - `dp[i][j-1] + 1` → insert into word1
```

**Base cases:** `dp[i][0] = i` (delete all), `dp[0][j] = j` (insert all)

> Example: word1 = "horse", word2 = "ros"
> "" | r | o | s
> "" | [ 0 | 1 | 2 | 3 ]
> h | [ 1 | 1 | 2 | 3 ]
> o | [ 2 | 2 | 1 | 2 ]
> r | [ 3 | 2 | 2 | 2 ]
> s | [ 4 | 3 | 3 | 2 ]
> e | [ 5 | 4 | 4 | 3 ] | answer = 3


```csharp
public class Solution
{
    public int MinDistance(string word1, string word2)
    {
        int m = word1.Length, n = word2.Length;
        var dp = new int[m + 1, n + 1];

        // Base cases
        for (int i = 0; i <= m; i++) dp[i, 0] = i;
        for (int j = 0; j <= n; j++) dp[0, j] = j;

        for (int i = 1; i <= m; i++)
        {
            for (int j = 1; j <= n; j++)
            {
                if (word1[i - 1] == word2[j - 1])
                {
                    dp[i, j] = dp[i - 1, j - 1]; // characters match
                }
                else
                {
                    dp[i, j] = 1 + Math.Min(
                        dp[i - 1, j - 1], // replace
                        Math.Min(
                            dp[i - 1, j],  // delete
                            dp[i, j - 1]   // insert
                        )
                    );
                }
            }
        }

        return dp[m, n];
    }
}
```

**Complexity:** O(m * n) time, O(m * n) space.

### Problem 4: Target Sum (LeetCode 494)

**Problem:** Given an integer array `nums` and a target, assign `+` or `-` to each element so their sum equals the target. Return the count of different assignments.

**Approach:** This transforms into a subset sum problem. Let P = sum of positives, N = sum of negatives. We need `P - N = target` and `P + N = totalSum`. So `P = (target + totalSum) / 2`. Count subsets summing to P.

**State:** `dp[j]` = number of ways to form sum `j` using elements considered so far

```csharp
public class Solution
{
    public int FindTargetSumWays(int[] nums, int target)
    {
        int totalSum = nums.Sum();

        // P = (target + totalSum) / 2 must be non-negative and integer
        if ((target + totalSum) % 2 != 0) return 0;
        if (Math.Abs(target) > totalSum) return 0;

        int subsetSum = (target + totalSum) / 2;

        // Count subsets with sum = subsetSum
        var dp = new int[subsetSum + 1];
        dp[0] = 1; // one way to make sum 0: pick nothing

        foreach (int num in nums)
        {
            // Traverse backwards to avoid using the same element twice
            for (int j = subsetSum; j >= num; j--)
            {
                dp[j] += dp[j - num];
            }
        }

        return dp[subsetSum];
    }
}
```

**Complexity:** O(n * subsetSum) time, O(subsetSum) space.

> **Note:** This is a 0/1 knapsack variant with counting. The backward loop ensures each element is used at most once.

### Problem 5: Interleaving String (LeetCode 97)

**Problem:** Given strings `s1`, `s2`, and `s3`, determine if `s3` is formed by interleaving `s1` and `s2` (maintaining relative order within each).

**State:** `dp[i][j]` = true if `s3[0..i+j-1]` can be formed by interleaving `s1[0..i-1]` and `s2[0..j-1]`

**Recurrence:**
- `dp[i][j] = (dp[i-1][j] && s1[i-1] == s3[i+j-1]) || (dp[i][j-1] && s2[j-1] == s3[i+j-1])`

**Base case:** `dp[0][0] = true`

```
Example: s1 = "aab", s2 = "axy", s3 = "aaxaby"

        ""    a     x     y
  ""  [ T     F     F     F ]
   a  [ T     T     F     F ]
   a  [ T     T     T     F ]
   b  [ F     F     T     F ]
                     ↑
        Not interleaving — answer would be false for this example
```

```csharp
public class Solution
{
    public bool IsInterleave(string s1, string s2, string s3)
    {
        int m = s1.Length, n = s2.Length;
        if (m + n != s3.Length) return false;

        var dp = new bool[m + 1, n + 1];
        dp[0, 0] = true;

        // Base cases: use only s1
        for (int i = 1; i <= m; i++)
            dp[i, 0] = dp[i - 1, 0] && s1[i - 1] == s3[i - 1];

        // Base cases: use only s2
        for (int j = 1; j <= n; j++)
            dp[0, j] = dp[0, j - 1] && s2[j - 1] == s3[j - 1];

        for (int i = 1; i <= m; i++)
        {
            for (int j = 1; j <= n; j++)
            {
                dp[i, j] =
                    (dp[i - 1, j] && s1[i - 1] == s3[i + j - 1]) ||
                    (dp[i, j - 1] && s2[j - 1] == s3[i + j - 1]);
            }
        }

        return dp[m, n];
    }
}
```

**Complexity:** O(m * n) time, O(m * n) space. Can be optimized to O(n) with a rolling row.

## Space Optimization for 2D DP

Many 2D DP problems only reference the current row and the previous row. You can collapse the table from O(m*n) to O(n):

```csharp
// Full 2D table
var dp = new int[m + 1, n + 1];
for (int i = 1; i <= m; i++)
    for (int j = 1; j <= n; j++)
        dp[i, j] = dp[i - 1, j - 1] + 1; // example

// Space-optimized with two rows
var prev = new int[n + 1];
var curr = new int[n + 1];
for (int i = 1; i <= m; i++)
{
    for (int j = 1; j <= n; j++)
        curr[j] = prev[j - 1] + 1;

    (prev, curr) = (curr, prev); // swap references
    Array.Clear(curr);
}
// Answer is prev[n]
```

> **Tip:** When the recurrence uses `dp[i-1][j-1]`, `dp[i-1][j]`, and `dp[i][j-1]`, a single row works if you save the diagonal value before overwriting it.

## Common Mistakes

1. **Off-by-one indexing** — 2D DP tables are often `(m+1) x (n+1)` to handle empty prefixes. Mix-ups between 0-indexed strings and 1-indexed DP are the top source of bugs.
2. **Wrong base case initialization** — Forgetting to set `dp[i][0]` or `dp[0][j]` correctly.
3. **Interleaving String: not checking lengths** — If `len(s1) + len(s2) != len(s3)`, return false immediately.
4. **Target Sum: ignoring parity/range checks** — If `(target + totalSum)` is odd or `|target| > totalSum`, the answer is 0.

## Interview Tips

- Draw the 2D table on paper with a small example. This makes the recurrence obvious and catches edge cases.
- When asked about space optimization, explain the rolling-row technique. It shows depth.
- For two-sequence problems (LCS, Edit Distance), the table structure is nearly identical — learn the pattern once.
- Mention that Edit Distance has real-world applications: spell checking, DNA alignment, diff utilities.

## Quiz

<details>
<summary>1. In Unique Paths, why is the first row and first column all 1s?</summary>

There is only one way to reach any cell in the first row (move right) or first column (move down). You cannot combine directions when restricted to a single row or column.
</details>

<details>
<summary>2. What is the key difference between LCS and Edit Distance recurrences?</summary>

In LCS, when characters match we add 1 to the diagonal (`dp[i-1][j-1] + 1`), and when they do not match we take the max of left and above. In Edit Distance, when characters match we copy the diagonal, and when they do not match we take the min of three neighbors plus 1 (replace, insert, delete).
</details>

<details>
<summary>3. How does Target Sum reduce to a subset sum problem?</summary>

By splitting elements into positives (P) and negatives (N), we get P - N = target and P + N = totalSum. Solving gives P = (target + totalSum) / 2. The problem becomes: count subsets summing to P.
</details>

<details>
<summary>4. In the space-optimized 2D DP, why do we swap rows instead of copying?</summary>

Swapping references is O(1), while copying is O(n). After computing the current row, the previous row is no longer needed, so we reuse its memory for the next iteration by swapping.
</details>

<details>
<summary>5. For Interleaving String, why must we check s1.Length + s2.Length == s3.Length first?</summary>

If the lengths do not add up, it is impossible for s3 to be an interleaving of s1 and s2 regardless of character arrangement. This early check avoids unnecessary computation and prevents index-out-of-bounds errors.
</details>
